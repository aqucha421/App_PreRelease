タッチ決済とQR決済の支払い記録を、できるだけ手入力なしで家計簿に残すための個人用PWAです。

このリポジトリは、Mac/Xcodeなしでまず自分だけ使えるプレリリースを目指す構成です。App Store公開ではなく、GitHub PagesなどのHTTPS公開先をiPhone Safariで開き、ホーム画面に追加して使います。

## できること

- QUICPay / iD / Suica / PASMO などのWallet取引を、iOSショートカット経由のURLで取り込む
- PayPay / 楽天ペイ / d払い / au PAY / メルペイの通知文や控え文から金額・店舗を半自動抽出する
- 店舗名やキーワードからカテゴリ分類する
- カテゴリ予算、月次レポート、未分類フィルタを確認する
- パスフレーズでローカル暗号化保存する
- JSONバックアップとCSV出力でデータを持ち出す
- PCブラウザ上の仮想iPhone画面で取り込み動作をデバッグする

## 重要な制約

- PayPayやカード会社の履歴を、アプリから直接取得するものではありません
- iOSショートカットが渡せる範囲の情報だけを取り込みます
- Walletの自動化トリガーは、カードやiOS設定によって取得できる値が変わります
- App Store公開やネイティブWallet API連携には、Mac/XcodeとApple Developer Programが必要です

## GitHub Pagesで使う

1. このフォルダの内容をGitHubリポジトリにpushする
2. GitHubでリポジトリを開く
3. `Settings` -> `Pages` を開く
4. `Deploy from a branch` を選ぶ
5. Branchは `main`、folderは `/root` を選ぶ
6. 表示された `https://ユーザー名.github.io/リポジトリ名/` をiPhone Safariで開く
7. Safariの共有ボタンから「ホーム画面に追加」を選ぶ

GitHubのファイル画面を共有するとログイン画面になりやすいので、共有するのはGitHub PagesのURLです。

## iPhoneショートカット連携

1. iPhoneでPWAを開く
2. アプリ内の「実機設定ガイド」を開く
3. 固定値テストURLをコピーする
4. ショートカットアプリでWallet取引のオートメーションを作る
5. `URL` アクションにコピーしたURLを貼る
6. 続けて `URLを開く` アクションを追加する
7. 固定値で追加できることを確認してから、`merchant` / `amount` / `provider` をショートカット入力に置き換える

## データ保護

明細は標準では端末内に保存されます。パスフレーズを設定すると、台帳・カテゴリルール・予算・広告表示設定を暗号化して保存します。

バックアップは `JSONバックアップ` で取得し、復元は `JSON復元` から行います。個人情報に近いデータなので、共有リポジトリにはバックアップJSONをpushしないでください。

## 公開前チェック

詳しい確認項目は `PUBLIC_PWA_CHECKLIST.md` にまとめています。

iPhoneで自分用プレリリース確認を進める場合は `REAL_DEVICE_PRERELEASE_GUIDE.md` を見ます。アプリ内にも「実機プレリリース確認」のチェック欄があります。

継続利用する場合は、アプリ内の「プレリリース運用」でバックアップ日、最新版確認、復旧手順を確認します。
