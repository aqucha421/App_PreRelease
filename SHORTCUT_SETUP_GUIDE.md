# ショートカット設定ガイド

## URL Scheme版

1. iPhoneで「ショートカット」アプリを開く。
2. 「オートメーション」を開く。
3. 新規オートメーションを作る。
4. 「取引」またはWallet関連のトリガーを選ぶ。
5. 対象カード/パスを選ぶ。
6. アクションで「URL」を追加する。
7. URLを以下の形で組み立てる。

```text
autoledger://transaction?provider=QUICPay&merchant=店舗名&amount=680
```

8. `provider`にはカード/パス名、`merchant`には取引の店舗、`amount`には金額を入れる。
9. アクションで「URLを開く」を追加する。
10. 実際に少額決済またはテスト用入力で動作確認する。

## PWA版の本番差し替え

GitHub Pagesで使うPWA版では、アプリ内の「本番ショートカット化」に表示されるURLを使う。

固定値URLで成功した後、以下の3つだけをショートカット入力値へ置き換える。

- `{{PROVIDER}}`: カード名、決済手段、または固定値
- `{{MERCHANT}}`: 加盟店名。取れない場合は確認待ちの店舗
- `{{AMOUNT}}`: 円記号なしの数字

必ず「URL」アクションの次に「URLを開く」アクションを置く。URLを作っただけではAuto Ledgerに渡らない。

## App Intents版

Xcode実装後、ショートカットに「Auto Ledgerに支払いを追加」が表示される想定。

1. Wallet取引トリガーを作る。
2. アクション「Auto Ledgerに支払いを追加」を選ぶ。
3. 金額、店舗、決済手段にShortcut Inputの各項目を割り当てる。
4. 実機で動作確認する。

## 注意

- Walletを通らないPayPay QR決済は、この取引トリガーでは拾えない。
- PayPayなどは通知文、レシート、CSVから半自動入力する。
- 取引情報の項目はカード会社、iOSバージョン、決済種別で差が出る可能性がある。

## トラブルシュート

### 決済しても追加されない

1. Auto Ledger PWAを開く。
2. 「ショートカット診断」の「テスト追加」を押す。
3. 999円のテスト取引が追加されるか見る。

追加される場合、アプリ側は動いている。ショートカット側で以下を確認する。

- オートメーションが有効
- すぐに実行になっている
- `URL` アクションの次に `URLを開く` がある
- URLがGitHub PagesのURLになっている
- Walletを通る決済で試している

追加されない場合、GitHub PagesのURLやPWA公開設定を確認する。
