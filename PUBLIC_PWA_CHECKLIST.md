## リポジトリ

- `index.html` がリポジトリ直下にある
- `app.js` / `styles.css` / `manifest.webmanifest` / `sw.js` が一緒にpushされている
- `icon.svg` / `icon-192.png` / `icon-512.png` がpushされている
- 個人の家計簿データ、JSONバックアップ、スクリーンショットをpushしていない

## GitHub Pages

- Pages URLが `https://ユーザー名.github.io/リポジトリ名/` 形式になっている
- GitHubのファイル閲覧URLではなく、Pages URLをiPhoneで開いている
- `index.html` が表示される
- 変更後に古い画面が出る場合は、アプリ内の「最新版に更新」または「キャッシュ削除」を使う

## iPhone Safari

- SafariでPages URLを開ける
- 共有ボタンから「ホーム画面に追加」できる
- ホーム画面アイコンがAuto Ledgerのアイコンになっている
- ホーム画面から起動すると、ブラウザのアドレスバーなしで表示される
- アプリ内の「実機プレリリース確認」で進捗をチェックできる

## 取り込みテスト

- アプリ内の「実機設定ガイド」に固定値URLが表示される
- 固定値URLの「テスト」で1件追加される
- ショートカットのWalletオートメーションから同じ固定値URLを開いて1件追加される
- 固定値テスト成功後、ショートカット入力値への置き換えを行う
- 追加されない場合は、アプリ内の「ショートカット診断」の最終受信と履歴を確認する
- 詳細手順は `REAL_DEVICE_PRERELEASE_GUIDE.md` を見る
- 原因が分からない場合は、アプリ内の「デバッグ情報コピー」で状態だけを控える

## データ保護

- パスフレーズで暗号化保存できる
- 再読み込み後に同じパスフレーズで復元できる
- JSONバックアップを1回取得できる
- JSON復元をテスト用データで確認できる
- バックアップファイルは公開リポジトリに置かない
- アプリ内の「プレリリース運用」でバックアップ日と復旧手順を確認できる

## 無料運用モデル

- スポンサー枠が明細を読まない説明になっている
- 広告表示のオン/オフが画面で確認できる
- 将来、広告を入れる場合でも店舗名・金額・メモ・カテゴリ・支払い時刻を広告側に渡さない
