# Auto Ledger v1 Spec

## One-line Concept

タッチ決済は自動、QR決済は半自動で、入力忘れを減らすローカル優先の家計簿アプリ。

## Target Users

- 家計簿を続けたいが手入力を忘れやすい人
- Apple Pay、QUICPay、iD、Suica、PASMOをよく使う人
- PayPayなどQR決済も使うが、完全自動でなくても入力の手間を減らしたい人
- 明細データを外部サービスへ預けたくない人

## Core User Flows

### 1. Walletタッチ決済の自動登録

1. ユーザーがiOSショートカットで取引オートメーションを設定する
2. Wallet対応カード/パスで支払う
3. ショートカットが金額、店舗名、カード/パス名を取得する
4. Auto Ledgerへ渡す
5. Auto Ledgerがカテゴリ分類して保存する

### 2. QR決済の半自動登録

1. ユーザーがPayPayなどで支払う
2. 通知文、レシート文、CSV明細のどれかをAuto Ledgerへ入れる
3. Auto Ledgerが金額と店舗名を抽出する
4. ユーザーが必要なら修正する
5. Auto Ledgerがカテゴリ分類して保存する

### 3. カテゴリ修正と学習

1. Auto Ledgerが店舗名からカテゴリを推定する
2. ユーザーが間違ったカテゴリを修正する
3. 同じ店舗は次回以降そのカテゴリに寄せる

### 4. 暗号化保存と復元

1. 取引データは端末内DBへ保存する
2. 秘密鍵はKeychainで保護する
3. バックアップ/エクスポート時は暗号化する
4. 復元時はユーザーの認証またはパスフレーズを求める

## Category Rules v1

- 交通費: 駅、JR、Suica、PASMO、メトロ、タクシー、バス
- 食費: コンビニ、スーパー、弁当、食堂、レストラン、食品
- カフェ: カフェ、喫茶、コーヒー
- 日用品: ドラッグ、薬局、日用品、ホームセンター
- 医療: 病院、クリニック、歯科
- 娯楽: 映画、書店、ゲーム、カラオケ
- 固定費: 電気、ガス、水道、通信、家賃
- 未分類: ルールに当たらない取引

## Privacy Promise

Auto Ledger v1は、ユーザーの取引明細を外部サーバーに送らず、端末内で保存・分類する。

外部に送信しないデータ:

- 店舗名
- 金額
- 取引日時
- 決済手段
- メモ
- カテゴリ修正履歴

将来同期機能を入れる場合も、同期は明示的なオプトインにし、サーバー側が明細を読めない暗号化方式を採用する。

## App Store Description Guardrails

使ってよい表現:

- Apple Walletの取引トリガーと連携
- ショートカット経由で支払い情報を登録
- PayPayなどQR決済は通知文やCSVから半自動入力
- 端末内保存
- ローカル分類

避ける表現:

- PayPayと完全自動連携
- 決済アプリから直接明細を取得
- すべての支払いを自動取得
- バックグラウンドで他アプリを監視
- 銀行レベルの完全な明細同期

## Acceptance Criteria

- ショートカット経由のサンプル入力で取引が作成される
- QR通知文から金額と店舗名が抽出される
- CSVから複数取引が取り込める
- カテゴリが自動分類される
- カテゴリを手動変更できる
- 手動変更が次回分類に反映される
- 取引は端末内に保存される
- ユーザーが全データを削除できる
- デモモードでApp Review担当者が主要機能を確認できる
