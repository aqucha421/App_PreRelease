    保存設計

    1. 端末内暗号化を基本にする 取引、店舗名、金額、時刻は個人情報に近いため、標準ではサーバーへ送らず端末内に保存します。
    2. 同期は明示的に選ぶ 複数端末同期は将来機能です。同期データも暗号化済みの塊として扱う方針です。
    3. 分類はローカル優先 店舗名ルール、手動修正、カテゴリ学習は端末内で完結します。外部AIへ明細を送りません。
    4. 共有するなら集計だけ 広告やバックアップに使う場合も、店舗名を含む生データではなくカテゴリ別集計などに限定します。

        本番ショートカット化

        固定値の次

        固定URLで追加できたら、下のURL内の3つの印をショートカット入力値へ差し替えます。

        {{PROVIDER}} カード名、決済手段、または固定値 QUICPay / Suica など
        {{MERCHANT}} Wallet取引の加盟店名。取れない場合は確認待ちの店舗でも使えます。
        {{AMOUNT}} 支払い金額。通貨記号なしの数字だけにします。
        1. 固定URLで成功させる まず固定URLテストで1件追加できる状態にします。
        2. URLアクションへ貼る 差し替え用URLをショートカットのURLアクションに貼ります。
        3. 3つの値を変数へ置換 {{PROVIDER}} / {{MERCHANT}} / {{AMOUNT}} をShortcut Inputの該当値に置き換えます。
        4. URLを開く 次のアクションに「URLを開く」を置きます。URLアクションだけでは送信されません。

        実機設定ガイド

        iPhone用
        1. ショートカットを開く iPhoneのショートカットアプリで「オートメーション」を開きます。
        2. Wallet取引トリガーを選ぶ 対象カードやSuica/PASMOを選びます。まずは固定URLでテストします。
        3. URLを作る 下の固定URLをコピーし、アクション「URL」に貼ります。
        4. URLを開く 次のアクションに「URLを開く」を追加します。URLだけでは記録されません。
        5. Wallet値に差し替える 固定URLテスト成功後、merchant / amount / provider をShortcut Inputの値へ差し替えます。
        追加されない場合は、オートメーションが有効か、「URL」アクションの次に「URLを開く」があるかを確認してください。

        このアプリの差分

        • 口座連携なし。金融機関ログイン情報を預かりません。
        • Walletタッチ決済はショートカットで即記録します。
        • PayPayなどQR決済は通知文やCSVから半自動で取り込みます。
        • 明細は端末内保存。外部AIへ送りません。
        • 無料運用を優先。広告を入れる場合も明細データを読ませません。
