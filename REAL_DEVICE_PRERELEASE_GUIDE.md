# Real Device Pre-release Guide

## 自分用プレリリース手順

`v0.9.2` から、アプリ内に「プレリリース手順パック」カードがあります。次の順で進めてください。

1. GitHubへpushする
2. GitHub Pages URLをiPhone Safariで開く
3. アプリ内の「最新版に更新」を押す
4. 初回セットアップを進める
5. 操作フロー確認を完了する
6. JSONバックアップを自分だけが開ける場所へ保管する
7. プレリリース判定にブロッカーがないことを確認する

手順パックのコピー内容には、店舗名や金額などの明細本文を含めません。

## 操作フロー確認カード

`v0.9.1` から、アプリ内に「操作フロー確認」カードがあります。固定URL追加、暗号化保存、JSONバックアップ、フィードバック記録、プレリリース判定までを、自分用プレリリース前の流れとして確認できます。

通し確認レポートには店舗名や金額などの明細本文を含めません。確認結果だけを控えたい場合に使ってください。

## 画面文言の整理

`v0.9.0` で、アプリ本体の静的文言とJavaScriptが動的に表示する文言を整理しました。家計簿ログ、月次レポート、診断、初回セットアップ、プレリリース判定、フィードバック記録の画面を、プレリリースで読める状態に近づけています。

## 実機フィードバックカード

`v0.8.8` から、アプリ内に「実機フィードバック」カードがあります。自分のiPhoneで使って気づいた不具合、使いにくい点、改善案、良かった点を端末内に記録できます。

高優先度の未解決メモがある場合は、プレリリース判定カードで注意として表示されます。改善メモはJSONバックアップに含まれるため、機種変更や復元時にも持ち越せます。

メモには、店舗名、金額、カード名などの明細本文をなるべく入れないでください。必要な場合も、原因調査に必要な最小限だけにします。

## プレリリース判定カード

`v0.8.7` から、アプリ内に「プレリリース判定」カードがあります。初回セットアップ、実機確認、固定URL受信、暗号化保存、JSONバックアップ、自己診断をまとめて確認できます。

判定は「ブロッカー」と「注意」に分かれます。ブロッカーが残っている場合は、自分用に使い始める前に解消してください。注意だけの場合は、内容を把握したうえで自分用プレリリースに進めます。

「判定レポートをコピー」は、店舗名・金額・メモなどの明細本文を含めず、進捗と状態だけをコピーします。

## 初回セットアップカード

`v0.8.6` から、アプリ右側に「初回セットアップ」カードがあります。自分のiPhoneで使い始める前に、次の順で確認してください。

1. 最新版に更新
2. 固定URLで1件追加
3. ホーム画面に追加
4. 暗号化保存を有効化
5. JSONバックアップを取る
6. 本番ショートカットを準備

固定URL受信、暗号化保存、JSONバックアップ、本番ショートカットURLコピーは、操作すると自動でチェックが進みます。ホーム画面追加はブラウザから正確に検出できないため、完了後に手動でチェックしてください。

JSONバックアップと明細データは個人情報に近いデータです。GitHub、公開共有リンク、誰でも見られるクラウドフォルダには置かず、自分だけが開ける場所で保管してください。

iPhoneでAuto Ledger PWAを自分用プレリリースとして確認する手順です。

## 事前準備

- GitHub Pages URLを用意する
- iPhone Safariで開く
- アプリ内の「最新版に更新」を1回押す
- 古い画面が出る場合は「キャッシュ削除」を押してから開き直す

共有するURLはGitHubのファイル画面ではなく、次の形式です。

```text
https://ユーザー名.github.io/リポジトリ名/
```

## 確認順

1. GitHub Pages URLで開く
2. Safariの共有ボタンから「ホーム画面に追加」する
3. ホーム画面のアイコンから起動する
4. アプリ内の「実機プレリリース確認」を開く
5. 「公開URLメモ」にPages URLを入力する
6. 「実機設定ガイド」の固定値URLでテスト追加する
7. 家計簿ログに1件追加されたことを確認する
8. iOSショートカットのWalletオートメーションから同じ固定値URLを開く
9. ショートカット診断の最終受信と受信履歴を確認する
10. パスフレーズを入れて暗号化保存する
11. 再読み込みして同じパスフレーズで復元する
12. JSONバックアップを取得する

## 固定値URLから本番ショートカットへ

固定値URLで成功したあとに行います。

1. アプリ内の「本番ショートカット化」を開く
2. 「差し替えURLをコピー」を押す
3. ショートカットの「URL」アクションへ貼る
4. `{{PROVIDER}}` をカード名、決済手段、または固定値へ置き換える
5. `{{MERCHANT}}` をWallet取引の加盟店名へ置き換える
6. `{{AMOUNT}}` を支払い金額へ置き換える
7. 金額は円記号なしの数字だけにする
8. 次のアクションに「URLを開く」を置く
9. 1回実行して、ショートカット診断の最終受信が更新されることを確認する

値が取れないカードや交通系ICでは、まず `provider=Suica` のように固定値を残し、`merchant` だけ「確認待ちの店舗」にする形でも使えます。

## 追加されない場合

- SafariでGitHub Pages URLを開いているか確認する
- GitHubのログイン画面が出る場合は、GitHub Pages URLではなくGitHubのファイル画面を開いている可能性が高い
- ショートカットで「URL」アクションの次に「URLを開く」アクションがあるか確認する
- Walletオートメーションが有効か確認する
- 最初はShortcut Inputを使わず、固定値URLのまま試す
- 固定値URLで成功してから `{{PROVIDER}}` / `{{MERCHANT}}` / `{{AMOUNT}}` を差し替える
- アプリ内の「ショートカット診断」の最終受信と履歴を見る
- 「自己診断を更新」を押して、確認が必要な項目を見る
- 「デバッグ情報コピー」を押して、状態だけを控える

## 完了報告

アプリ内の「実機プレリリース確認」でチェックを入れたあと、「確認結果をコピー」を押すと、確認状況をテキストで控えられます。

## デバッグ情報

「デバッグ情報コピー」は、以下のような状態だけをコピーします。

- アプリバージョン
- URL形式
- 明細件数
- 受信履歴件数
- 保存状態
- 実機確認の進捗
- Service WorkerやlocalStorageの対応状況

店舗名、金額、メモなどの明細本文は含めません。

## 運用ルール

自分用プレリリースとして使う間は、次の運用にします。

- 週1回、またはショートカット設定を変えた直後にJSONバックアップを取る
- GitHubへpushした後は、iPhone側で「最新版に更新」を押す
- 古い画面が出る時は「キャッシュ削除」を押してから開き直す
- 復旧手順はアプリ内の「プレリリース運用」からコピーして控える
- バックアップJSONは公開リポジトリに置かない

## 復旧手順

データが消えた、または別端末で使いたい場合:

1. GitHub Pages URLをSafariで開く
2. 「最新版に更新」を押す
3. 必要なら「キャッシュ削除」を押す
4. 「JSON復元」を押す
5. 保存していたバックアップJSONを選ぶ
6. 明細件数、カテゴリ、予算を確認する
7. パスフレーズを入れて「解除して保存」を押す
8. 固定値URLで1件追加されるか確認する

## 注意

- 取引データやバックアップJSONは個人情報に近いので、GitHubへpushしない
- PayPayやカード会社の履歴を直接取得しているわけではない
- Wallet取引は、iOSショートカットが渡せる範囲の情報だけ記録する
